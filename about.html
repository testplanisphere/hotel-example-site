解説

研修メニューを考える方や指導する方向けにより詳しい解説を記載します。このサイトは前述の通りテスト自動化の初心者向けの研修に使うことを想定して作成しています。

そのためブラウザテスト自動化の際に引っ掛かりがち、嵌まりがちな要素をある程度意図的に配置しています。

共通

エラーメッセージはエラーが発生した入力欄の下部に表示されます。表示タイミングは画面によって異なります。何度か各画面を触ってもらえれば仕様は把握できると思います。

ログイン画面

シンプルな画面です。エラーメッセージはボタンの押下時に表示されます。未入力の場合、メールアドレス・パスワードの両方もしくは一方が登録内容と異なる場合にエラーとなります。

この画面のみid属性を多く付与してあります。研修ではまずこの画面の操作から始めるのが良いと思います。

  • <input type="email">
  • <input type="password">
  • <button type="submit">
会員登録画面

エラーメッセージはボタンの押下時に表示されます。入力必須項目とそれ以外の項目があります。長さ指定のある項目では境界値のテストができます。

dateのようなツールから扱いづらい項目は無理に初心者向けの研修に含める必要はないでしょう。

  • <input type="text">
  • <input type="radio">
  • <input type="checkbox">
  • <input type="email">
  • <input type="password">
  • <input type="tel">
  • <input type="date">
  • <select><option>
マイページ画面

マイページ本体は表示のみの画面です。ログインまたは登録の画面と組み合わせて一本のテストシナリオができると思います。

新規登録したユーザの場合、「アイコン設定」「退会する」のボタンが有効になります。

アイコン設定では画像のアップロードができます。登録した画像はマイページで表示されます。rangeとcolorはツールから扱いづらいため研修で使う際には注意してください。

退会のボタンではJavaScriptのダイアログが連続で表示されます。

  • <input type="file">
  • <input type="range">
  • <input type="color">
  • JavaScriptダイアログ confirm()
  • JavaScriptダイアログ alert()
宿泊プラン一覧画面

トップの一つを除き、プランの情報はAjaxによって非同期で取得され表示されます。またログイン状態とユーザの会員ランクによって表示されるプランが変わります。

  • Ajax (Fetch API)
  • 新規ウィンドウ表示 (target="_blank")
宿泊予約画面

入力内容によって合計金額が計算されて表示されます。組み合わせが多いためテスト技法の練習にもいいかもしれません。

この画面用の計算ロジックは単体テストがあります。

  • Ajax (Fetch API)
  • jQuery UI Datepicker
  • <input type="checkbox">
  • <input type="text">
  • <input type="number">
  • <input type="email">
  • <input type="tel">
  • <select><option>
  • <textarea>
  • <iframe>
宿泊予約確認画面

予約画面で入力した内容の確認ができます。確定時にはダイアログがアニメーション（フェードイン）付きで出現します。

  • 非JavaScriptダイアログ Bootstrap Modal
